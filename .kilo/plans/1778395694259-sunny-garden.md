# Course-based Training System

Implement a system where users must mark all sections of the Staff Handbook as "read" before they can access the SSD training quiz.

## Goals
- Prevent users from skipping the handbook.
- Track progress for each user individually.
- Provide a clear visual indicator of completion.

## Technical Plan

### 1. Database Schema Update
- Create a new MongoDB collection `training_progress`.
- Each document should track:
    - `userId`: Discord User ID.
    - `completedSections`: An array of section IDs from the handbook.
    - `handbookCompleted`: Boolean flag.
    - `lastUpdated`: Timestamp.

### 2. Handbook Progress API
- Create a new API route `pages/api/training/progress.js` to:
    - `GET`: Fetch the current user's completed sections.
    - `POST`: Toggle a section as read/unread and update the `handbookCompleted` flag when all sections are checked.

### 3. Handbook UI Enhancements (`pages/training/handbook.js`)
- Add checkboxes next to each section in the contents list and the main content areas.
- Sync checkbox state with the `training_progress` API.
- Disable/hide the "Quiz" or "Training" link until `handbookCompleted` is true.
- Add a progress bar at the top of the handbook.

### 4. Quiz Access Guard (`pages/training/index.js`)
- Before loading the training iframe, check the user's progress via the API.
- If `handbookCompleted` is false, redirect the user to `/training/handbook` with a notification explaining that they must read the handbook first.

### 5. Validation & Testing
- Verify that progress persists across sessions.
- Ensure that marking all sections as read correctly unlocks the quiz.
- Test the redirection logic for users who attempt to bypass the handbook.
