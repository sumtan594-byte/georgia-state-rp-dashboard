# Plan: Migrate Transcripts from GitHub to MySQL Database

## Overview
Replace the current GitHub-based transcript storage with MySQL for both the Management Bot (writer) and the Dashboard (reader). Transcripts get a UUID instead of the `__` delimited filename.

```sql
CREATE TABLE IF NOT EXISTS transcripts (
    id VARCHAR(36) PRIMARY KEY,
    type VARCHAR(50) NOT NULL,
    owner_id VARCHAR(255) NOT NULL,
    channel_name VARCHAR(255) NOT NULL,
    closed_at DATETIME NOT NULL,
```

### 1.3 Create `utils/transcriptDB.js`
New file with a single function `saveTranscriptToDB(data)`:
- Accepts: `{ type, ownerId, channelName, closeReason, openerTag, openReason, claimedBy, claimedByTag, staffRequestReason, htmlContent, closedAt }`
- Generates a UUID v4 using Node's `crypto.randomUUID()`
- Performs `INSERT INTO transcripts (...)` 
- Returns the UUID

Option A (cleaner): Modify `createTranscript` in `utils/ticketUtils.js` to also return the metadata:
